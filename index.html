Команды управления

Основные команды для работы со скриптами в вашем мире (максимум 10 файлов на мир):

КомандаОписание
/code create <имя>Создать новый file скрипта в текущем мире
/code book <имя>Выдать книгу для редактирования кода выбранного файла
/code saveСохранить код из книги в руке (название книги должно быть именем файла)
/code reloadПерезагрузить все скрипты в текущем мире
Важно: Для сохранения файла /code save вы должны держать в руке книгу, полученную через команду book. Название книги автоматически устанавливается как имя файла — не меняйте его.

Структура скрипта

Чтобы код заработал, он должен быть обернут в функцию с аннотацией @event. Вы можете писать несколько функций в одной книге.

Пример 1: Базовое приветствие (с коротким алиасом)

@event
void onJoin(JoinWorldEvent e) {
    player.msg("&aДобро пожаловать в наш мир!")
}

Пример 2: Отмена разрушения и выдача блока травы

Пример использования события ломания блока. Код проверяет, сломал ли игрок траву: если да, то отменяет разрушение (блок остаётся стоять) и выдаёт 1 блок в инвентарь с помощью алиасов.

@event
void onBreak(BlockBreakEvent e) {
    // Отменяем событие с помощью короткого алиаса
    event.cancel()
    
    // Проверяем, является ли сломанный блок травой
    if (block_type == "GRASS_BLOCK") {
        // Выдаем игроку 1 блок с помощью короткого .getInv()
        player.getInv().addItem(ItemStack("GRASS_BLOCK", 1))
    }
}

Пример 3: Табличка-телепорт (на спавн)

@event
void onSpawnSign(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK") {
        if (block_type == "OAK_SIGN") {
            player.sendMessage("&aВы успешно телепортированы на спавн!")
            player.teleport(Location(0.5, 64.0, 0.5))
        }
    }
}

Пример 4: Табличка для очистки инвентаря

Правый клик по дубовой табличке полностью стирает все вещи из инвентаря игрока.

@event
void onClearSign(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK") {
        if (block_type == "OAK_SIGN") {
            // Полностью очищаем все вещи игрока
            player.getInv().clear()
            player.msg("&cВаш инвентарь был полностью очищен!")
        }
    }
}

Пример использования PickupItemEvent

Запрещает подбирать абсолютно любые лежащие на земле предметы, если игрок находится в режиме приключения (ADVENTURE).

@event
void onItemPickupCheck(PickupItemEvent e) {
    if (player.gamemode == "ADVENTURE") {
        // Отменяем подбор предмета
        event.cancel()
    }
}

Данные из событий и игрока

Эти локальные переменные автоматически доступны внутри функций и подстраиваются под контекст запущенного события:

Общие данные игрока (Доступны везде):

player.name — Никнейм игрока, запустившего событие.
player.gamemode — Текущий режим игры ("SURVIVAL", "CREATIVE", "ADVENTURE", "SPECTATOR").
player.level — Текущий уровень опыта игрока.
player.exp — Точный чистый прогресс по шкале текущего уровня (от 0.0 до 1.0).
player.food — Уровень индикатора сытости/голода игрока (от 0 до 20).
player.saturation — Текущий уровень насыщенности (скрытая шкала сытости).
player.health — Количество текущего здоровья игрока (например, 20.0).
player.isFlying — Летит ли игрок в данный момент (true/false).
player.isAllowFlight — Разрешен ли полет на сервере для этого игрока (true/false).
player.isOnGround — Находится ли игрок твердо на блоке/земле (true/false).
player.isGliding — Парит ли игрок на элитрах (true/false).
player.isGlowing — Имеет ли игрок эффект контурного свечения (true/false).
player.isSprinting — Находится ли игрок в состоянии быстрого бега (true/false).
player.isSneaking — Находится ли игрок в приседе/на шифте (true/false).
player.locale — Языковой пакет клиента в нижнем регистре (например, "ru_ru", "en_us").
player.ping — Текущий замер сетевого пинга игрока в миллисекундах.
player.x, player.y, player.z — Округленные до целого числа координаты местоположения игрока.
player.handItem — Название предмета в главной руке игрока (если рука пуста — "AIR").
player.handItem.cmd — Custom Model Data предмета из NBT (если присутствует).
player.handItem.name — Кастомное отображаемое имя предмета (если переименован).

Системные данные текущего мира (World):

world.name — Строковое название текущего загруженного подмира.
world.playercount — Число active игроков, находящихся в этом мире в данный момент.

Переменные контекста событий:

event_type — Тип текущего события ("CHAT", "DROP_ITEM", "INTERACT_ENTITY", "PICK_ITEM", "TOGGLE_SNEAK", "TOGGLE_SPRINT").
block_type — Материал блока (для Interact/Place/Break, например "STONE", "GOLD_BLOCK").
block_x, block_y, block_z — Координаты блока в мире (числа).
action — Тип клика (актуально для InteractEvent: "RIGHT_CLICK_BLOCK", "PHYSICAL").
message — Текст сообщения из чата (Доступно только в ChatEvent).
item_type — Материал выброшенного/подбираемого предмета (Доступно в DropItemEvent и PickupItemEvent).
entity_type — Тип существа, по которому кликнули (для InteractEntityEvent, например "VILLAGER", "ITEM_FRAME").
frame_item — Если кликнули по рамке, вернет тип лежащего в ней предмета (если пустая — "AIR").
is_sneaking — Итоговое состояние приседания после изменения (true/false, доступно только в PlayerSneakEvent).
is_sprinting — Итоговое состояние бега после изменения (true/false, доступно только в PlayerSprintEvent).

Продвинутые текстовые методы

Для любых строковых данных (например, переменной message) можно использовать встроенные методы логической проверки, а также вычисление длины и подстроки:

  • .startsWith("текст") — Истинно, если строка начинается с этого фрагмента.
  • .endsWith("текст") — Истинно, если строка оканчивается этим фрагментом.
  • .contains("текст") — Истинно, если фрагмент найден в любом месте строки.
  • .length() — Возвращает число, содержащее общую длину строки в символах.
  • .substring(beginIndex) или .substring(beginIndex, endIndex) — Обрезает и возвращает часть строки по указанным индексам (поддерживает динамические аргументы-переменные и имеет автоматическую встроенную защиту от крашей при выходе за реальные границы строки).

Пример 1: Проигрывание звука по сообщению

@event
void onChatCheck(ChatEvent e) {
    if (message.startsWith("@пигстеп")) {
        player.playSound("music_disc.pigstep", 1.0, 1.0)
    }
}

Пример 2: Получение длины строки и создание подстроки

@event
void onSubstringDemo(ChatEvent e) {
    var len = message.length()
    if (len > 5) {
        // Безопасно вырезаем первые 5 символов сообщения
        var firstFive = message.substring(0, 5)
        player.msg("&eПервые 5 символов: %firstFive%")
    }
}

Переменные, Рандом и Циклы

Глобальные: доступны всегда во всех файлах этого мира (пишутся без var).
Локальные: доступны только внутри выполнения конкретной функции (пишутся с var).

@event
void myLogic(InteractEvent e) {
    // Глобальная переменная для всех
    total_clicks = total_clicks + 1
    
    // Случайное число от 1 до 10 (Возвращает ЧИСЛО)
    var chance = random(1, 10)
    
    // Корректное числовое сравнение без кавычек
    if (chance == 7) {
        player.msg("&eДжекпот! Вы попали на число 7! Всего кликов в мире: %total_clicks%")
    }
}

Умный выбор: random.choice()

Метод random.choice(аргументы...) случайным образом выбирает один из текстовых вариантов, автоматически подставляя значения переменных, если они указаны внутри процентов.

@event
void onRandomGreet(JoinWorldEvent e) {
    // Рандомим сообщение с автоматическим раскрытием %player_name%
    var text = random.choice("Привет %player_name%!", "Дарова!", "meow %player_name%!")
    player.msg("&2%text%")
}

Поиск игрока мира: world.getPlayer()

Динамически находит игрока в текущем мире по его имени и инжектирует в локальную область памяти переменные с его текущими параметрами, используя имя переменной в виде префикса.

@event
void onFindPlayer(ChatEvent e) {
    if (message.startsWith("@check ")) {
        var nameArg = message.substring(7)
        
        // Находим игрока
        var target = world.getPlayer(nameArg)
        
        if (target != none) {
            player.msg("&aИгрок %nameArg% онлайн! Его здоровье: %target.health%, уровень: %target.level%")
        } else {
            player.msg("&cИгрок %nameArg% не найден в этом мире!")
        }
    }
}

🔄 Глобальный цикл по игрокам мира (ЭКСПЕРИМЕНТАЛЬНО)

Конструкция forEachPlayer { ... } позволяет выполнить вложенный блок кода для каждого игрока, находящегося в текущем мире исполнителя. Внутри тела цикла автоматически генерируются изолированные свойства с префиксом target., отражающие параметры итерируемого игрока в реальном времени.

@event
void onLevelBuff(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" && block_type == "DIAMOND_BLOCK") {
        
        player.msg("&aЗапуск сканирования мира... Раздача регенерации новичкам!")

        // Запускаем цикл по всем игрокам мира
        forEachPlayer {
            // Проверяем уровень опыта конкретного перебираемого игрока (target)
            if (target.level == 1) {
                // Накладываем Регенерацию I на 10 секунд на цель
                target.effect("REGENERATION", 10, 1)
                target.msg("&aВам выдана Регенерация, так как ваш уровень равен 1!")
            }
        }
    }
}

Работа с Инвентарем, Кулдаунами и Задержками

Пример 1: Выдача предмета при входе в мир

Когда новый игрок заходит в ваш мир, скрипт автоматически добавляет в его инвентарь 16 единиц хлеба.

@event
void onGift(JoinWorldEvent e) {
    // Выдать предмет через короткий алиас (Тип, Количество)
    player.getInv().addItem(ItemStack("BREAD", 16))
}

Пример 2: Обмен 64 алмазов на незерит на нажимной плите

Если игрок наступает на каменную нажимную плиту (действие PHYSICAL), скрипт проверяет наличие 64 алмазов. Если их хватает — изымает их и выдаёт 1 незеритовый слиток.

@event
void onDiamondExchange(InteractEvent e) {
    // Проверяем, что наступили на блок
    if (action == "PHYSICAL") {
        if (block_type == "STONE_PRESSURE_PLATE") {
            
            // Посчитать количество предмета у игрока
            var diamonds = player.getInventory().count("DIAMOND")
            
            // Проверка что хватает алмазов для обмена
            if (diamonds >= 64) {
                player.getInv().clear("DIAMOND", 64)
                player.getInv().addItem(ItemStack("NETHERITE_INGOT", 1))
                
                player.msg("&aВы успешно обменяли стак алмазов на незерит!")
            }
        }
    }
}

Пример 3: Перо Скорости (+ Механика Кулдаунов)

Использование кулдауна для ограничения частоты применения предметов. При клике пером, игрок получает Скорость II на 7 секунд, а на все перья вешается перезарядка на 10 секунд.

@event
void onSpeedFeather(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" || action == "RIGHT_CLICK_AIR") {
        // Проверяем, держит ли игрок перо через системную переменную
        if (player.handItem == "FEATHER") {
            // Накладываем Скорость II на 7 секунд (7 сек * 20 тиков переведутся автоматически)
            player.effect("SPEED", 7, 2)
            player.msg("&bАртефакт активирован! Скорость выдана на 7 секунд.")
            
            // Включаем кулдаун на предмет FEATHER на 10 секунд
            player.cooldown("FEATHER", 10)
        }
    }
}

Пример 4: Перо отталкивания / прыжка (Leap Feather)

При использовании пера игрок ускоряется в направлении взгляда (подбрасывается вперед), воспроизводится звук взлета летучей мыши и устанавливается перезарядка в 5 секунд.

@event
void onLeapFeather(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" || action == "RIGHT_CLICK_AIR") {
        if (player.handItem == "FEATHER") {
            player.push(2.5)
            player.playSound("entity.bat.takeoff", 1.0, 1.0)
            player.cooldown("FEATHER", 5)
        }
    }
}

Пример 5: Железная автоматическая пушка (Iron Cannon)

Кликая железной конской броней, игрок быстрой очередью выпускает стрелы со звуком выстрела арбалета. Кулдаун составляет всего 0.15 секунды.

@event
void onIronCannon(IronCannon e) {
    if (action == "RIGHT_CLICK_BLOCK" || action == "RIGHT_CLICK_AIR") {
        if (player.handItem == "IRON_HORSE_ARMOR") {
            player.launch("ARROW", 1.0)
            player.playSound("item.crossbow.shoot", 1.0, 1.0)
            player.cooldown("IRON_HORSE_ARMOR", 0.15)
        }
    }
}

⏱️ Планирование асинхронных задержек времени

Методы wait(секунды) и waitTicks(игровые_тики) приостанавливают проход текущих строк кода, полностью изолируя и копируя текущее состояние памяти localVars, и переносят оставшиеся инструкции в планировщик Bukkit.

@event
void onDelayedTrap(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" && block_type == "OAK_BUTTON") {
        player.msg("&cЛовушка взведена! Бегите...")
        
        // задержка 5 секунд (в тики переведется автоматически)
        wait(5)
        
        // Выполнение продолжится планировщиком с сохраненными переменными
        player.playSound("entity.generic.explode", 1.0, 1.0)
        player.msg("&4БУМ!")
    }
}
Примечание по Кулдаунам: Проверять кулдаун вручную через if в коде не нужно. Когда на предмет наложен кулдаун, Майнкрафт автоматически блокирует клики, и это событие просто не будет вызываться движком, пока перезарядка не пройдет!

Изменение Хранилищ и Выдача Лута

Умные методы для работы с контейнерами (сундуки, бочки, шалкеры):

  • world.copy(x1,y1,z1, x2,y2,z2) — копирует содержимое контейнера с координатной точки 1 в точку 2.
  • player.giveLoot(x,y,z) — опустошает сундук/бочку на координатах и выдает весь его лут напрямую в инвентарь игроку с полным сохранением NBT (зачарования, прочность, кастомные имена). Если инвентарь забьется — остатки красиво упадут под ноги.

Пример: Наполнение сундука для Голодных Игр

@event
void onChestRefill(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" && block_type == "STONE_BUTTON") {
        // Копируем из тайного сундука (0,66,-4) в игровой сундук (10,65,10)
        world.copy(0, 66, -4, 10, 65, 10)
        broadcast("&eЛут в сундуках на арене обновлен!")
    }
}

Тайтлы, Режимы Игры, Эффекты и Звуки

Вы можете отслеживать игровой режим игрока через свойство player.gamemode, накладывать эффекты зелий и проигрывать кастомные звуки.

@event
void onButtonClaim(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" && block_type == "OAK_BUTTON") {
            
        // Проверяем режим игры (Всегда возвращает СТРОКУ в верхнем регистре)
        if (player.gamemode == "SURVIVAL") {
            player.sendTitle("&c&lХей!", "&7Только для креатива", 10, 60, 10)
            
            // Воспроизведение звука (Звук, Громкость, Питч)
            player.playSound("entity.villager.no", 1, 1)
        }
    }
}

Управление эффектами и звуками:

player.effect("REGENERATION", 10, 1)           // Наложить Регенерацию I на 10 сек
player.clearEffects()                         // Снять вообще все эффекты с игрока

player.stopSound("ambient.cave")               // Выключить определенный звук игроку
player.stopAllSounds()                         // Полностью заглушить все звуки

Изменение пространства и Спавн сущностей

Метод world.fill(Материал, X1, Y1, Z1, X2, Y2, Z2) позволяет мгновенно заполнять или очищать область блоков. Также доступны новые методы спавна/удаления живых существ.

Метод world.spawn("моб", Location(x, y, z), hasAI, isSilent) спавнит моба в указанных координатах, можно указать если ли ии у моба а также будет ли издавать звуки.

@event
void onTrapButton(InteractEvent e) {
    if (action == "RIGHT_CLICK_BLOCK" && block_type == "STONE_BUTTON") {
        // Заполняем область лавой
        world.fill("LAVA", 10, 60, 10, 12, 60, 12)
        
        // Спавним злого моба на ловушке и сохраняем его ID в глобальную переменную
        boss_mob = world.spawn("WITHER_SKELETON", Location(11, 61, 11), "false", "true")
        
        broadcast("&cАктивирована ловушка! Опасность!")
    }
}

Удаление существ:

// Уничтожает сущность в мире, которая была записана в переменную boss_mob
world.remove(boss_mob)

Действия игрока и Таблица Алиасов

Интерпретатор одинаково успешно компилирует как развернутый Java-синтаксис, так и быстрые сокращения:

Остальные методы API:

broadcast("Текст")                            // Объявление на весь текущий мир (приватно для игроков этого мира)
player.setGamemode("SURVIVAL")                 // Смена режима игры ("SURVIVAL", "CREATIVE" и др.)

player.setHealth(20)                        // Установить HP
player.setFood(20)                            // Установить уровень голода
player.setSaturation(5)                     // Установить насыщенность

player.clearTitle()                           // Убрать тайтлы с экрана

// Телепортация игрока на локацию
player.teleport(Location(10.5, 64.0, 10.5))

Текст и Плейсхолдеры

В любых выводимых строках поддерживается динамический парсинг данных:

  • %player_name% — Имя инициатора события.
  • %player_gamemode% — Текущий геймод игрока.
  • %любая_переменная% — Значение вашей глобальной или локальной переменной.
  • Цветовые коды через амперсанд & (например: &a — зелёный, &e — жёлтый, &l — жирный шрифт).
